FlySafe: Centralised Pre-Flight Briefing Dashboard

  • Group project | 4 Month
  • Role: Competitor analysis, Designing interview, Thematic analysis, Journey flow

Overview

Pre-flight briefings are an essential part of flight operations, yet crew managers face significant challenges with fragmented systems and outdated workflows. The FlySafe project aimed to address these issues by designing a centralised dashboard that improves efficiency, streamlines communication, and reduces the overall preparation time for briefings.

Design Framework: The Double Diamond Approach

We followed the Double Diamond Design Process to ensure the project was both user-focused and solution-driven:

● Discover Phase

In this phase, we focused on understanding the challenges and opportunities. As a result The need for a centralized, hierarchical, and communication-friendly platform became evident. Here are our Research Methods:

Interview
  • Conducted with four crew managers from different airlines to understand their workflows and frustrations.
  • Key Insight: Managers had to log into 7–8 different systems to gather briefing information, often resorting to manual note-taking.
  • Quote: “I wish it was just one system where I could access all the information!”

The thematic analysis of the interviews

Observational Study
  • Shadowed a crew manager as they prepared a briefing.
  • Findings: The process took 40–60 minutes and was plagued by inefficiencies, including redundant logins and delays in accessing critical updates.
Competitor Analysis
  • Evaluated tools like NAVBLUE and SkyLife Pro.
  • Identified gaps: None of the existing tools offered a centralised, AI-powered dashboard tailored to pre-flight briefings.

● Develop Phase

This phase focused on ideating and prototyping.

Information Architecture (IA):
  • Developed IA using card sorting and tree testing.
  • Iterated the IA based on SME (Subject Matter Expert) feedback:
    • Improved terminology (e.g., replacing "Galley" with "Area of Responsibility").
    • Refined paths for clarity (e.g., simplifying safety-related categories).

Happy Path:

To map the ideal user flow, we developed a Happy Path Diagram that outlines the journey in three phases:

  1. Awareness: The user is notified that their briefing materials are ready.
  2. Engagement: The user prepares the briefing, saving or editing recommendations as needed.
  3. Execution: The user conducts the briefing seamlessly using the prepared materials.

Prototyping:

Low-Fidelity Prototypes: Designed in Figma to validate the layout and navigation. (First iteration)

Key Features of the Prototype:
  • Digital Notes and Checklists: Replace manual methods with intuitive tools.
  • AI-Driven Recommendations: Automate task assignments and safety question selection.
  • Centralized Dashboard: Seamlessly integrate data from multiple sources.

● Deliver Phase

This phase focused on Testing and Iterations.

Remote Moderated Usability Testing:
  • Conducted with SMEs to evaluate the first prototype.
  • Key Findings:
    • Preparation time reduced from 40–60 minutes to 20 minutes.
    • Feedback emphasized the need to remove redundant steps and improve data hierarchy.
  • This usability test led to the second iteration.
Second Prototype
  • Removed unnecessary steps (e.g., scrollable information page).
  • Improved data hierarchy, prioritizing safety and security details.
  • Added catering details to the service brief.
  • Added automated calculations for working hours and service brief updates.
Remote Unmoderated Usability Testing:
  • Conducted with 10 SMEs using Optimal Workshop.
  • Metrics: Usability score of 79/100, 90% task success rate, and minimal misclicks.

Final Solution

The FlySafe Dashboard integrates key features:

  • Centralised Information: All flight-related data in one accessible platform.
  • Real-Time Updates: Adapt to last-minute changes efficiently.
  • Automated Tools: Delegate tasks and generate reports seamlessly.
  • User-Friendly Design: Simplify workflows with digital tools for note-taking, checklists, and presentations.

Impact:

  • Reduced briefing preparation time by 50%.
  • Positive feedback from SMEs on simplicity and efficiency.
